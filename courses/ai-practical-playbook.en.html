Introduction

Why Demo built this course

Over the past six months, models and AI workflows have kept changing — ChatGPT/OpenAI, Claude, Gemini, image & video generation, connectors and agent tools have all shipped new updates. Each shift changes how you prompt, which tool to pick, and how AI fits your workflow.

At Demo, data and insight are the product. Keeping the whole team — BD, Account, Research, Marketing, HR, Tech — current on these shifts is a direct competitive advantage.

This isn't a book of "prompt spells." It explains how AI works so each person can build the workflow that fits their team's work. Updated when a notable new model or tool ships.

You'll be confident to

  • → Tell when to use pretrained knowledge, when to use web search, when to use deep research.
  • → Pick the right model/tool for the task type: text, search, research, image, video, audio, data, automation — no "one size fits all".
  • → Use AI to brainstorm, challenge, and draft — while keeping a real human voice and avoiding hollow output.

Before we start

AI changes workflows, not people

Before we get into the technical content, one thing needs to be said clearly. With AI moving as fast as it is, some people are excited, some people are anxious. Both reactions are normal — but Demo's view of AI is straightforward:

① Doesn't replace skilled people

AI doesn't replace people who are good at the craft. Real people understand the client, understand the market, read cultural nuance — AI doesn't have any of that. AI is just a tool that extends your reach.

② Replaces repetitive workflows

AI replaces repetitive, low-leverage workflows: report rollups, data cleaning, first-draft emails, mock visual concepts, mention tagging. The "time-consuming but undifferentiated" work — so you can spend those hours on what actually matters.

③ Demoian value still holds

A Demoian's value still lives in strategic thinking, client understanding, and the ability to create impact. AI handles the rough work, you handle the refined work. What sets the best users apart is judgement: knowing when to trust the output and when to push back.

How Demo sees it

Ask the right question. Drop "is AI going to take my job". Ask instead: "how many low-value hours per week can I free up to do work that actually matters".

This isn't an "AI survival skills" course. It's a guide to using AI to do the work you're already good at better — and to expand what one person at Demo can do.

Module 01 · Foundations

Effective information retrieval with AI

AI has 3 main ways to retrieve information: pretrained knowledge, real-time web search, and deep research. Understand each so you pick the right tool.

Lesson 1.1

AI Novice vs AI Power User

A clear pattern across teams using AI: two people with the same access to ChatGPT/Claude/Gemini, but their results differ by 5–10x. What decides it is how you approach the tool, not any "magic words".

AI Novice

The non-power approach

  • → One short question, expecting a perfect answer
  • → No context (job, audience, desired format)
  • → Trusts 100% of the output, no verification, no follow-up
  • → Uses one model for everything
  • → Gives up after the first failed try
AI Power User

The effective approach

  • → Treats AI like a new coworker: needs a brief, needs feedback loops
  • → Always gives context: who you are, who the audience is, what the output is for
  • → Iterates 3–5 times, refines piece by piece
  • → Picks the right model for the task (reasoning vs fast vs research)
  • → Verifies output against evidence, especially numbers and data
Mental Model

"AI is a smart intern who doesn't know the company yet"

An outstanding Stanford intern just joined Demo: extremely smart, reads fast, writes well — but doesn't know who your client is, what the brand voice sounds like, or how Demo structures reports. Your job is to brief them. The better the brief, the better the output.

At Demo

Power user mindset by department

▶ BD / Sales Write proposals faster

Novice: "Write a proposal for client X."

Power user: "I'm a BD at Demo Media. The client is a mid-tier FMCG dairy brand, ~500M VND/quarter budget, currently interested in Social Listening. They just lost Q4 market share. Write a 4-slide draft proposal: (1) a summary of their problem based on a hypothesis I'll verify, (2) how SocialPulse solves it, (3) deliverables and 8-week timeline, (4) investment range. Tone: confident but no over-promising."

▶ Research / Insight Stronger brief = deeper insight

Novice: "Analyze mentions about brand X for me."

Power user: "Below are 200 mentions from SocialPulse about brand X in Nov 2025. Group them by: (1) sentiment (positive/negative/neutral), (2) topic cluster (product, service, price, promo), (3) influencer tier discussing the brand. Find 3 emerging issues that could turn into a crisis in the next 2 weeks. Return JSON with a clear structure."

▶ Tech / Engineering Brief code agents properly

Novice: "Write me a dashboard."

Power user: "Stack: Next.js 15 App Router + tRPC + Tailwind + shadcn/ui. Dashboard shows 4 KPI cards (Total mentions, Sentiment %, Top channel, Top influencer), 1 line chart over 30 days, 1 word cloud. Data from the existing tRPC endpoint dashboard.getKpi (interface attached). Code should be modular — one component per card, with loading skeletons and error boundaries."

Takeaway

Power users win by briefing AI properly: give context, iterate, verify, pick the right tool. "Magic words" decide nothing. Treat AI as a smart intern who doesn't know the company yet.

Lesson 1.2

Pretrained Knowledge — What AI already knows

Every AI model has a training cutoff — the last day it "read" the internet. Everything before that date it has "memorized"; everything after, it doesn't know (unless web search is on).

AI is strong at
  • ✓ Academic knowledge, history
  • ✓ Industry best practices (marketing, code, etc.)
  • ✓ Concepts, definitions, frameworks
  • ✓ Code, formulas, languages
  • ✓ Writing, general brainstorming
Be careful with
  • ⚠ Specific numbers (always verify)
  • ⚠ Names of people, specific companies
  • ⚠ Quotes / citations
  • ⚠ Laws, regulations (may have changed)
  • ⚠ Prices, product specs
Don't use pretrained for
  • ✕ Events after the cutoff date
  • ✕ Today's news
  • ✕ Stock prices, exchange rates
  • ✕ Schedules, event calendars
  • ✕ Client/brand-specific information
Hallucination — when AI "makes things up"

When AI doesn't know the answer, it can produce convincing-sounding but wrong information. This is hallucination. Every number, name, link, and quote AI produces must be verified — especially in client reports.

Sample prompt · Check for hallucination 
The answer above contains specific data (numbers, names, years). For each data point, rate it on this scale:
- HIGH CONFIDENCE: definitely correct (common knowledge)
- MEDIUM: probably correct, worth verifying
- LOW: uncertain, may have changed or be fabricated
- HALLUCINATED: you don't actually know — you just guessed

Return a 3-column markdown table: Data | Confidence | Reason
Takeaway

Use pretrained knowledge for concepts and frameworks; always verify numbers, names, and events; know your model's training cutoff.

Lesson 1.3

Web Search — When you need to look online

Most modern models (ChatGPT, Claude, Gemini) have a web search feature: AI runs a real Google/Bing query, reads the top results, then synthesizes. You get an answer with sources instead of just AI memory.

Turn web search on when
  • → Asking about recent news or events
  • → You need specific, sourced numbers
  • → Market or competitor research
  • → Looking up latest versions of tools, frameworks
  • → Asking about specific people, companies, or products
No web search needed
  • → Writing, brainstorming, refactoring code
  • → Explaining a concept
  • → Translation, format conversion
  • → Summarizing text you already pasted
  • → Roleplay, creative content
At Demo

Web search use cases by team

BD

Pre-pitch client research: who leads the company, recent news, recent campaigns.

Research

Compare industry benchmarks (CTR, engagement rate) from 2025–2026 sources instead of pretrained data.

Marketing / Content

Find trending topics, hashtags, and newsjacking opportunities within 24 hours.

Tech

Check breaking changes in new libraries, security advisories, release notes.

HR

Latest industry salary benchmarks by level and city.

Account

Track client PR moves to proactively suggest actions in the weekly report.

Takeaway

Web search = questions that need freshness + a citation. Pretrained = questions about concepts + skills. Don't use web search for everything — it's slower and sometimes noisier.

Lesson 1.4

Source quality — How to evaluate sources

AI search reads a handful of web sources and synthesizes an answer for you. Which sources it uses, which search engine, and how many results it reads can vary by platform, account tier, and timing. So don't just trust the summary: open the original links, check the publish date, check the author, and prioritize primary sources before they go into a report.

Tier 1 · Trustworthy
  • • Reports from Nielsen, Kantar, eMarketer
  • • Government sites (gov.vn, sec.gov)
  • • Peer-reviewed scientific journals
  • • Tier-one press: Reuters, Bloomberg
  • • Official company blogs, IR docs
Tier 2 · Verify further
  • • Vietnamese mainstream press
  • • Industry blogs with named authors
  • • Expert Medium / Substack writers
  • • Wikipedia (as a starting point)
  • • Reddit, Quora (for ideas, not numbers)
Tier 3 · Be careful
  • • SEO content farms
  • • AI-generated blogs (the AI loop)
  • • Aggregator news with no original source
  • • Old forums, Facebook comments
  • • PR pieces not labeled as PR
Sample prompt · Make AI rate its own sources 
Find numbers on [the Vietnam E-commerce market over the last 12-24 months] and:
1. List 5 DIFFERENT sources you used (full links).
2. Classify each source into Tier 1/2/3.
3. Mark which are primary sources and which are secondary.
4. If sources disagree on a number → state it clearly and propose which figure is more trustworthy, with reasoning.
5. End with a 1-paragraph executive summary that can be pasted into a Demo ECI report.
Warning · Empty-source loop

When AI searches a popular topic, more and more top results are AI-generated content sourced from other AI. This loop creates "fake consensus." How to check: tell AI to find the primary source and don't accept blog roll-ups.

Takeaway

Every claim in a Demo client report needs source + period + sample size. Make AI sort into Tier 1/2/3, dig to the primary source, and don't accept aggregated numbers.

Lesson 1.5

Deep Research Mode — When to dig deep

Deep Research is a mode where AI spends more time finding sources, cross-reading them, and writing a structured report. It's slower than regular chat, but useful when the question needs many sources, multiple angles, and clear citations.

Chat
A few seconds
Little to no sourcing
Web Search
Fast
A few web sources
Deep Research
Noticeably longer
Many queries, many sources

When is Deep Research worth it?

Worth it: industry landscape reports for clients, market sizing, competitive analysis, due diligence before pitching a new vertical, strategic theses.
Not worth it: simple questions, captions, one-fact checks, idea brainstorming — regular chat is faster.
Deep Research Prompt · Industry landscape 
Goal: write a landscape report on [Beauty & Personal Care on TikTok Shop Vietnam over the last 12-24 months] for the Demo Media BD team pitching an FMCG client.

Deep research requirements:
1. Market size for the most recent year, growth rate, and projection if there is a trustworthy source.
2. Top 10 active brands (both global and Vietnam local).
3. 3 viral content trends (UGC type, hashtag, format).
4. Which KOL/KOC tier is currently most effective (sub vs mega).
5. CPV, CPM, engagement rate benchmarks for this category.
6. Insight into consumer behavior that differs from Shopee/Lazada.

Deliverable:
- Executive summary, 200 words
- ~2,500-word body report split into the 6 sections above
- Comparison table of the top 10 brands
- Full reference list (prioritize Tier 1/2 sources)
- Mark any claim without a source as "[needs source]"
Pro Tip · The better the brief, the more valuable the deep research

Deep Research runs longer than regular chat, so a bad brief easily wastes your time. Before you hit "Deep Research," always: (1) review the prompt with AI in regular chat ("is this prompt clear enough?"), (2) ask AI to propose an outline before digging, (3) confirm the outline, then run it.

Takeaway

Deep Research suits landscape reports, industry analysis, and due diligence. Brief the full outline, source quality tier, and deliverable format before hitting start. Don't use it for 5-minute tasks.

Lesson 1.6 · Lab

Model comparison — Picking the right AI for the task

There is no "best model" — only "the right model for the task." Because models change fast, the table below is a way to choose by workflow, not a fixed leaderboard. For anything important, test 2-3 models on the same real brief from your team, then decide.

How to keep your own model map current
  • Official docs first: OpenAI / Anthropic / Google docs and release notes for models, context, file support, and data policy.
  • Benchmarks second: Artificial Analysis, LMArena, LiveBench are only a starting reference — they don't replace your own testing.
  • Test on real tasks: same brief, same file, same rubric. Score on: on-brief, fewer errors, easy to verify, cost, policy.
  • Decide by workflow: prioritize the tool your team is licensed for, that has a safe connector, and that is easy to share with colleagues.

Compare: which model sounds most natural? Most creative? Most on-brief? Make notes for next time, and pull in your lead or 1–2 teammates to try again and lock in a choice.

Module 1 takeaway

Three modes of information retrieval: Pretrained · Web Search · Deep Research. Pick based on how fresh the information needs to be and how deep you need to dig. Each model has its strengths — don't lock in to one model for every task, especially when the market is updating every week.

Module 02 · Thinking Partner

AI as a thinking partner

AI is strongest when you treat it as a teammate thinking with you, not a vending machine for answers. This module covers brainstorming, context, reasoning, writing — and how to keep AI from flattering you.

Lesson 2.1

Brainstorming — Expanding ideas with AI

AI can produce 20–50 ideas a minute. Your brain can't. The split: AI handles quantity, you handle quality — pick, combine, push further.

The "3-Round Brainstorm" recipe

  1. 1
    Quantity: "Give me 30 ideas about [X]. Diverse, including wild ones. One sentence each."
  2. 2
    Pivot: Pick 3 ideas you like. "Take these 3 and pair them up — give me 6 hybrid ideas."
  3. 3
    Depth: Pick 1 winner. "Dig into this one: risks, resources, timeline, KPIs, who'd push back and why."
Sample prompt · Campaign brainstorm for Marketing 
I need to brainstorm content angles for the Tet 2026 campaign for brand [X — mid-priced carbonated soft drink, target Gen Z].

Round 1: give me 25 content angles. Diversity criteria:
- 5 emotional angles (nostalgia, connection, etc.)
- 5 angles on Tet culture in the North/South/Central regions
- 5 humor / meme angles
- 5 sustainability / social angles
- 5 "wild" angles no brand has done yet

Format: number + 1-sentence tagline + 1-sentence execution note.
At Demo
BD / Sales

Brainstorm 20 cold-email opening hooks per vertical — pick 3 that fit the client's industry.

Account

Brainstorm 15 ways to present the same data point — pick the one that fits the client contact's personality.

Marketing

Brainstorm a content calendar — 30 ideas/month, group by theme, pick 12.

HR

Brainstorm 20 team-building activities, filter by budget and team size, present 3 to leadership.

Design

Brainstorm 20 visual concepts / key visuals for a brief, mock 3 quickly, then present.

Tech / Product

Brainstorm 15 ways to fix an architecture bug, weigh trade-offs, pick the approach for the PRD.

Takeaway

Brainstorming = AI handles quantity, you handle quality. Run 3 rounds: expand → combine → deepen. Don't stop at the first answer.

Lesson 2.2

Context — A power user's number one weapon

80% of the gap between weak AI output and excellent output is context. AI can be an expert, but it can't read your mind. Good context = AI knows who you are, who the audience is, what the output is for.

The C.R.A.F.T framework
C
Context — the situation
R
Role — who you are
A
Audience — the reader
F
Format — the structure
T
Tone — the voice
Before · no context
Write an apology email to the client for the delayed report.
After · full C.R.A.F.T
[C] I'm an Account Manager at Demo Media, client A is the CMO
of a large FMCG brand, 2-year relationship, slightly formal.
The weekly report is 1 day late due to a technical issue with
the data source, outside our control.
[R] Account Manager, confident but accepting responsibility.
[A] CMO — busy, skims, hates excuses.
[F] Vietnamese email, 80–120 words, one clear next step.
[T] Professional, no over-apologizing, propose a solution
in the email.

Write the draft. Don't say "very sorry" or "sincerely sorry"
more than once.
Demo Context Block · reusable template

Save in a note app, paste at the start of every work-related AI chat:

[BACKGROUND]
I work at Demo [subsidiary name], my role is [role].
The company: a data & marketing-insight agency, core products
[SocialPulse / EcomPulse / CommunityHub / SurveyVoice / etc.].
Client I'm working on: [industry], tier [Top X / SME / startup].
Communication culture: direct, evidence-based, low on buzzwords,
"consultant" tone rather than "agency."

[OUTPUT PREFERENCE]
- Natural, fluent Vietnamese.
- Every claim backed by evidence or marked [assumption].
- Clear structure: headings + bullets + tables.
- No clichés, no "in the 4.0 era," no "digital era."

Keep this context for every following answer.
Takeaway

A strong prompt = Context + Role + Audience + Format + Tone (C.R.A.F.T). Save a Demo context block and paste it every time. You'll save hours of refining.

Lesson 2.3

AI Desktop Apps & MCP — When AI works on your machine

Web chat is enough for most day-to-day work. Desktop apps, coding agents, and connectors are worth it when you need AI to read context from another app, work with files/repos on your machine, or call a tool it has been granted access to. Not every tool can read files, send email, or query a database; the real capability depends on the app, the plan, the connector, and the access your admin grants.

Web chat

Regular chat

Text dialogue, file upload, image generation. Simple, no setup.

Desktop app

ChatGPT / Claude / Gemini app

Handy for hotkeys, screenshots, voice, small files, and working alongside other apps. Exact features vary by platform.

MCP & Agents

Claude Code · Codex · IDE agents

AI reads a repo, proposes file edits, runs commands/tests when allowed, or calls a tool via a connector. Review carefully before it writes files, sends requests, or runs automation.

What is MCP?

Model Context Protocol is an open standard for AI apps to connect to external servers/tools. A server can expose documents, data, or actions like reading a file, finding a ticket, or calling an API. AI can only do what the server supports and what the user/admin has allowed. For internal data, start with read access, limit the scope, and have a human review the output.

New trend

AI agents & automation — from chat to a tool-using workflow

CONNECTOR / APP
Read the right source

AI reads Drive/Slack/CRM/GitHub through granted access instead of manual copy-paste. Needs admin approval and access control.

SCHEDULE
Run on a schedule

Daily scan, weekly digest, monthly report draft. Good for repeated work, but output still goes through a human review before it reaches a client.

ACTIONS
Create files / run code

AI cleans a CSV, builds a chart, drafts a deck, updates a template. Suits tasks with a clear checklist and a way to verify the result.

→ Don't automate too early. Build a prompt template first, use it 5-10 times, measure ROI and common errors, then promote it to a workflow with a connector/schedule.
At Demo · Concrete workflows
▶ Account / Account Manager

If the Gmail/Drive connector has been approved by IT: ask AI to read only the granted folder/thread, summarize the 5 biggest issues of the week, point to the original email/file, then draft a weekly update for the account to review before sending.

▶ Research / Insight Analyst

An agent/code tool working on a prepared export folder: reads the CSV, runs the analysis script, generates draft charts, writes the insight section. The owner still checks the sample, the formulas, the charts, and the wording before it goes into the deck.

▶ Tech / Engineering

A coding agent in the repo: "read the project guideline, look at the PRD in docs/, implement endpoint X, write tests, explain the diff." Use a mode that requires approval on the real repo; the owner still reviews the code and runs the tests before merging.

▶ HR / Admin

With an approved tool: use AI to summarize CVs against the JD and create a draft shortlist. Avoid uploading PII to a public chat; don't let AI reject candidates on its own before an HR person reviews the criteria and the results.

Security · Important for Demo

When granting connector/MCP access to client data, give it only the minimum it needs. Start read-only where possible. Don't let an AI agent write/delete production files without a guardrail and approval. Don't paste PII into a public chat. A recurring workflow must have an owner, logs, and a way to stop it when output goes wrong.

Takeaway

Desktop apps, connectors, and agents are only worth it when they cut copy-paste or help verify a repeated workflow. Start small: read-only, narrow scope, a human reviewer. Allow stronger actions only once it's stable.

Lesson 2.4

Reasoning Models — When you need AI to "think hard"

Regular models answer fast. Reasoning / thinking mode in the major tools spends more processing steps before answering, so it suits tasks with complex constraints. In return it's slower, and not needed for short writing, simple summaries, or quick brainstorming.

Turn reasoning on when
  • → Logic problems, math, complex code
  • → Multi-step planning (workflow, automation design)
  • → Decisions with many constraints and trade-offs
  • → Data analysis with open-ended questions
  • → Debugging hard bugs
No reasoning needed
  • → Writing a caption, a short email
  • → Simple translation, summarization
  • → Quick brainstorm, need many options
  • → Quick Q&A
  • → Casual chat
Reasoning prompt · Trade-off analysis 
Think carefully before answering.

Situation: Demo ECI must pick one of two paths for a new client (a Top 20 VN cosmetics brand, 1.2B VND/quarter budget):

Option A — SocialPulse focus: deep social listening + sentiment + crisis monitoring.
Option B — EcomPulse focus: e-commerce performance + share of voice on Shopee/TikTok Shop.

Trade-off analysis:
1. List 5 decision criteria (KPIs, ROI, retention, upsell potential, fit with the client's current pain point).
2. For each criterion, rate A vs B (1–5).
3. Which criterion matters most for this client — and why?
4. Final recommendation.
5. Biggest risk of that recommendation.

Return a criteria table, scores, the key evidence/assumptions, a recommendation, and a confidence level. You don't need to spell out all of your hidden reasoning.
Pro Tip

For questions with many checking steps (math, logic, code, planning) → reasoning is usually worth a try.
For creative / subjective questions (writing, design) → reasoning can make output stiffer. A regular model is often more creative.

Takeaway

Reasoning mode is worth it when you need analysis with criteria, evidence, assumptions, and confidence. You don't need to turn it on for everything. For short tasks, a fast model is usually enough.

Lesson 2.5

Sycophancy — When AI flatters you

AI is trained to make users happy. But "happy" and "right" don't always overlap. Show AI an idea and hint you like it → AI praises it. Ask "this idea's bad, right?" → AI agrees. This is sycophancy — the most common ailment of today's AI.

Risky for Demo

You're writing a proposal and ask AI "is my approach okay?" — it's easy to get back "very solid, professional, on track" — even when the proposal is weak. You pitch the client, lose the deal, never know why.

Questions that trigger sycophancy
  • "Is this idea good?"
  • "Is my plan solid?"
  • "What do you think of this great idea?"
  • "Is this code clean?"
Questions that force AI to push back
  • "5 reasons this idea could fail."
  • "Critique this plan like a senior consultant."
  • "Who would push back, and why?"
  • "Find 3 logic bugs in this code."
Anti-sycophancy template · paste at the top of every serious chat 
Rules for this conversation:

1. DO NOT praise my ideas to make me feel good.
2. If you see a logic gap, missing information, or a wrong assumption — say it directly, no "however" or "you might want to consider."
3. When I propose a plan or idea, default to playing a senior reviewer at a competing agency — they'll point out the weak spots to win the deal.
4. If I push back weakly, hold your ground when you're right. Don't change your view just because I didn't like the answer.
5. When you're uncertain, say "uncertain" — don't make things up.

Confirm you understand, then I'll paste what needs review.

Concrete Demo applications

BD pitch

"Play the client's CMO. Read this proposal and list 5 reasons you would NOT sign."

Research report

"Find 3 logical fallacies or cherry-picks in the report below."

Code PR

"Play a senior staff engineer. List every issue you'd flag in this code review."

Takeaway

By default, AI flatters you. Power users actively force pushback: use a "senior critic" role, ask "5 reasons this fails," paste an anti-sycophancy block before any review.

Lesson 2.6

Writing with AI — Avoid hollow writing

By default, AI writes like "the average of the internet": full of buzzwords, clichés, every paragraph opening with "In today's digital age...". Clients and managers spot it in 5 seconds.

Words to avoid
in the 4.0 era digital era constantly evolving remarkable improvement optimize leverage synergy unlock delve into it's important to note in conclusion — em dash in every sentence "it's not X — it's Y" "not just X, but Y"
Demo voice · what to aim for
  • → Short, active sentences
  • → Data first, commentary after
  • → Specific headings: "Sales -12% Q4" instead of "Business situation"
  • → Every claim → evidence or [assumption] tag
  • → One idea per paragraph
  • → Skip the filler: "very important" → "important"
Style guide block · paste before any writing request 
VOICE: Demo — consultant, evidence-driven, no clichés.

HARD RULES:
- Clear Vietnamese. When keeping English terminology, don't inline-translate if the meaning is already clear.
- Average sentence ≤ 22 words. Avoid clauses-within-clauses.
- DO NOT use: "in the 4.0 / digital era," "the era of," "leverage," "unlock," "delve into," "significant," "remarkable," "very [adj]," "extremely [adj]," "it could be said that," "undeniably."
- DO NOT open with "In the context of..." or "In the era of...".
- DO NOT close with "In summary" / "In conclusion" / "Overall."
- Em dash (—) max once per paragraph, no spamming.
- NO contrastive cliché: "It's not X — it's Y" / "not just X, but Y". This is a classic ChatGPT tell; readers spot it instantly and react badly.

STRUCTURE:
- Lead with the takeaway. The first sentence is the conclusion.
- Each point: 1-sentence claim → 1-sentence evidence/number → 1-sentence so-what.
- Specific headings, with data if available.

STOP when done. No "hope this helps."
The "Voice Capture" trick

Want AI to write in your voice? Paste 3–5 paragraphs you've written, with this prompt: "Analyze my style: sentence patterns, vocabulary, paragraph structure, words I use often, words I avoid. Return a detailed style guide. Then write [new topic] in that style." Save the style guide for reuse.

BD email

Cold emails open with an insight about the client, not "Hope this finds you well."

Account report

Every slide opens with a finding/conclusion, not "This section covers...".

Marketing copy

Facebook captions open with a specific sentence, not emoji + rhetorical question.

HR job description

Drop "passionate, energetic, dynamic team" → describe what one day on the job actually looks like.

Takeaway

Hollow writing can ruin a proposal faster than a not-quite-polished data table. Always paste the style guide block, capture your voice, re-read before sending — a 10% rewrite is enough to lose the "AI smell."

Lesson 2.8

Managing context & memory — working with AI over long sessions

C.R.A.F.T handles a single prompt. But real work rarely finishes in one line: you go back and forth 20–30 turns on a report, a proposal, a piece of code. The further that thread (one continuous chat) runs, the more the AI forgets constraints you set up front, contradicts itself, or turns vague. Steering the whole conversation, not just the opening line, is the habit that takes you from decent to fluent.

Why long chats go fuzzy

Every model can only hold so much of the conversation in view at once — its working memory for the chat, called the context window. As a thread grows, the model tracks details from early on less and less reliably, and anything past that limit drops off entirely. This is often called context rot: quality slips as the conversation piles up, even on models with a huge context window.

Signs a thread is overloaded
  • → The AI repeats itself, or contradicts your original instructions
  • → It forgets constraints you set (format, tone, banned words)
  • → Output gets blander, stops gripping specifics
  • → You keep re-pasting the same context
The power-user reflex
  • → One task = one thread. New task, new chat
  • → Quality drops? Reset. Don't try to "heal" the old thread
  • → Hand off with a summary, not the whole history
  • → Recurring context goes into memory/projects, not re-pasted

Reset or continue?

Continue the same thread: you're building on the same artifact (one email, one slide, one function) and need the AI to remember decisions just made.
Start a new chat: you're switching to a different task; quality is sliding; the thread is full of trial-and-error branches; or you want the AI to "forget" a wrong direction it's been stuck on.
Technique · Summary handoff — carry state into a new chat 
Before I open a new chat, write a tight "state summary" I can paste over:

- GOAL: (one line)
- DECIDED: the key decisions so far (bullets)
- CONSTRAINTS / STYLE: format, tone, banned words (bullets)
- CURRENT DRAFT: paste verbatim if short, or describe the state
- REMAINING: next steps (bullets)

Write enough that another AI with no history can pick it up right away.
Persistent memory

Memory & Projects — stop re-pasting context every time

Most major assistants now have some form of memory (remembering a few things across chats) and projects or workspaces (shared files + instructions across many chats); the exact names and behavior vary by tool. That's where recurring context lives: your role, brand voice, the report format your team uses — instead of pasting your C.R.A.F.T block every time (see Lesson 2.2).

Data note: don't put Tier 3 material (NDAs, PII, client numbers) into memory; it lingers and can surface in other chats. Review what memory has stored periodically and delete what you don't need (see Lesson 4.1).
At Demo
Account

One thread per client deliverable. Before closing it, ask for a state summary and save it to your notes so next week picks up without losing the thread.

Research

Reset the chat between two different analyses. Don't let insight on brand A bleed into brand B in the same thread.

Marketing

Put brand guidelines + tone into a project/memory once; each campaign only needs its own brief, no re-pasting everything.

Tech

Store repo conventions + stack in a project. When a coding thread gets tangled, summarize "what's fixed, what's still broken" and open a clean thread to keep debugging.

Module 2 takeaway

A good prompt is a sentence skill; managing context is a session skill. One task per thread, reset when it goes fuzzy, hand off with a state summary, and push recurring context into memory/projects — and keep sensitive data out of both.

Tasks AI does well with images

  • ✓ Extract data from charts and graphs

  • ✕ Reading sensitive info (ID cards, passports)

Lesson 4.1

AI Safety & Data Privacy — Don't leak client data

Demo works with hundreds of brands: NDAs, raw mention data, internal financials, employee PII. One misplaced prompt can send client data to an unapproved system. Before using AI, classify your data and pick the right account/tool.

🟢 Tier 1 · Public

Paste freely

  • Public news articles
  • Brand descriptions already published
  • Public social posts
  • Generic document templates
🟡 Tier 2 · Internal

Anonymize first

  • Demo internal reports
  • Draft discussion guides
  • Pricing templates (numbers removed)
  • Mention data with brand names redacted
🔴 Tier 3 · Confidential

Never paste into public AI

  • Client NDAs / contracts
  • Raw client data with brand names
  • Salary, HR records, CVs
  • API keys, credentials, passwords
3-step rule before pasting
  1. Ask: "If this data hits the news tomorrow, does someone get hurt?" If yes → Tier 3, stop.
  2. Anonymize: Swap "Vinamilk" → "Brand A (FMCG dairy)", swap employee names → "Employee_01", redact sensitive numbers.
  3. Pick the right tier of tool: Tier 3 only goes through a company-approved tool/account with clear business/enterprise terms on data usage, retention, audit, and admin controls. Free/Personal plans are NOT enough.
How to read a policy fast
  • Business/API/Enterprise usually commit to not using input/output for training by default, but you still need to read the retention section and the admin controls carefully.
  • Free/Personal policies can differ by product, country, activity setting, feedback, or human review.
  • Unknown AI tools are not safe just because the landing page says "private". Get IT/Legal review before putting client data in.
Prompt: Auto-anonymize before processing
Before analyzing, read the text below and:
1. List ALL potentially confidential information: brand names, person names, specific numbers, contract amounts, emails, phone numbers.
2. Create an anonymized version: brands → [BRAND_A], people → [PERSON_01], exact numbers → "~approximately X".
3. Ask me to confirm before starting the analysis.

[Paste data here]

  • Pasting a raw SocialPulse export with real brands → leaks client insight.

💡 Takeaway

AI doesn't know which data is allowed to leave — you have to tell the difference. Before pasting: tier it → anonymize → pick the right tool/account. When in doubt, ask your lead or IT/Legal first.

Lesson 4.2

Advanced Prompt Patterns — 5 techniques that double output quality

C.R.A.F.T is the base framework. When you need to push output from "passable" to "usable after review," layer in the 5 patterns below. Each one solves a specific problem.

Pattern 01

Few-shot — Teach AI by example

When you need AI to follow a specific format/style, don't describe it — give 2–3 input → output examples. AI mimics the format nearly perfectly.

Example for the Account team
Write client email subject lines in Demo style. Here are 3 examples:

Input: Weekly social listening report for brand X Output: [SocialPulse] Brand X Weekly Pulse — 3 insights for action

Input: Send Tet campaign pricing proposal Output: [Proposal] Tet 2026 Campaign — Investment & Expected Impact

Input: Reschedule meeting due to conflict Output: [Reschedule] [Topic] — 3 proposed new slots

Now generate one for: "Notify client about an emerging crisis"
Pattern 02

Reasoning Scaffold — Make AI analyze with structure

Instead of just writing "think step by step", give a clear checklist: which criteria to consider, the order of analysis, and how short the rationale in the output should be. With a reasoning model, you don't need to force it to show all of its hidden reasoning; just ask for conclusion + evidence + assumptions + confidence level.

Example for the Research team
Analyze whether this viral post (15k shares) is organic or seeded.

Evaluate against this checklist:
1. Engagement pattern: is the like/share/comment ratio unusual?
2. Comment quality: are the first 20 comments copy-paste, same wording, same timing?
3. Timing: did shares grow gradually and naturally, or spike in clusters?
4. Account pattern: are there many new accounts, little history, or the same network?

Output:
- Conclusion: organic / likely seeding / not enough data
- The 3 strongest pieces of evidence
- Assumptions or data still missing
- Confidence 0-100%

[Paste data]
Pattern 03

Role + Persona — Cast AI as an expert

"You are a Senior Strategic Planner with 15 years at WPP, having run campaigns for Unilever, Heineken..." — the more specific, the more expert the tone. Avoid generic roles like "you are an assistant."

Pattern 04

XML Structuring — Separate inputs clearly

When a long prompt has multiple parts (context + data + instruction + example), wrap them in XML tags. AI parses them much more accurately than flat prose. Especially effective with Claude.

Standard template for complex tasks
<context> Demo Media, social listening agency. Client: [BRAND]. Audience report: Gen Z 18-24 HCMC. </context>

<data> [Paste 50 top-engagement mentions] </data>

<task> Identify top 3 conversation themes and sentiment for each. </task>

<format> Markdown table with columns: Theme | Volume | Sentiment | Example quote </format>
Pattern 05

Output Constraint — Force exact format

Specify exactly what output you want: word count, format, no preamble, no markdown, or a specific JSON schema. AI will stop with the "Sure, here is..." padding.

Example for Tech / Data team
Return EXACTLY the following JSON, with no explanation:
{
  "sentiment": "positive" | "neutral" | "negative",
  "confidence": float 0-1,
  "topics": [string],
  "actionable": boolean
}

Input mention: "[paste mention]"
💡 Takeaway

These 5 patterns aren't meant to be used together — pick by need: Few-shot for format, Reasoning scaffold for analysis, Role for an expert perspective, XML for long prompts, Output Constraint for integration. Combining 2–3 of them in the right places is usually enough to raise quality noticeably.

Lesson 4.3

Integrate AI into team workflow — From individual to enterprise scale

Personal AI skill alone isn't enough. When you share prompt templates, measure time saved, and document workflows — that's when Demo really benefits. The 4 steps below have worked across many teams.

01

Identify repetitive tasks (≥ 3h/week)

Each team member lists their 5 most time-consuming tasks of the week. Aggregate → pick the top 3 tasks with a recurring pattern (report templates, email drafting, meeting summaries, data formatting...). These are your first automation targets.

02

Build & test a standard prompt template

One person on the team (the AI champion) builds the prompt using C.R.A.F.T + the right patterns. Test on 3–5 real cases, tune. Document: expected input, expected output, edge cases that don't work.

03

Share & train — workflow doc + buddy session

Save the prompt to the team's Notion/Confluence. Run a 30-minute buddy session: AI champion demos, each member runs it once, Q&A. Add to the onboarding doc for new hires.

04

Measure & iterate — use numbers, don't guess

Before/after: task completion time, number of required revisions, reviewer satisfaction. Review quarterly. Workflows no longer useful → archive. High-ROI workflows → graduate to a tool with UI (build an app, Zapier, n8n...).

✓ Needs an audit if the skill has runnable code

Custom GPT / Gem / Skill = "AI with knowledge available when you open a chat." An agent workflow goes further: AI can use connectors/tools, read multiple sources, run code, create files, or run on a schedule. This tier is useful once the task is stable and has a clear owner.

  • → Google Drive, SharePoint, Slack, CRM, ticket systems

  • → No manual paste/export if a safer connector exists
SCHEDULED WORK
Run on a schedule

  • → Weekly competitor digest

CODE / FILE ACTIONS
Do the repetitive steps
  • → Clean a CSV, render a chart, generate a deck draft
  • → Update a template/report against a checklist
  • → Test the file output and permissions before scaling
When to consider it: tasks that are routine + repetitive + have a clear checklist + have someone responsible for review. Build process for any assistant (works on any platform — 3 steps)
  1. Identify a recurring task: what task gets the same context pasted ≥ 3 times/week? That's a candidate.
  2. Draft + test on 5 real cases: build a minimum viable assistant. Test 5 different cases. Tune the instructions until 4 out of 5 outputs are send-ready.
  3. Share + version: share the link/Skill with the team. Other members use it → feedback → update. Define an owner for maintenance.
⚠️ Security across platforms
  • Knowledge files: don't upload contracts, NDAs, or raw client data with brand names to a public Custom GPT/Gem — anonymize first.
  • Custom action / API key: put it on the backend, don't embed keys in GPT instructions — instructions can be extracted.
  • Skills from external sources: treat as software install — audit scripts, file-fetch URLs, bash commands carefully.
  • Share scope: default to internal team only. Public only after a review confirms there's no sensitive data.
  • Enterprise plans: use a data-not-trained plan (Claude Enterprise, ChatGPT Enterprise, Gemini for Workspace Enterprise) for assistants holding internal knowledge.
